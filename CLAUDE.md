## 概要

ローカルで動作する工数管理・日報生成ツール。PCとスマートフォンのどちらからも入力でき、Notionデータベースをバックエンドとして同期する。

- **メインファイル**: `index.html`（旧 index_v2.html をリネーム済み）
- **動作環境**: ブラウザで直接開くだけで動作（サーバー不要）

---

## Notionデータベース構成

APIプロキシ: `https://notion-proxy.itjyouhou.workers.dev`（CORS回避用 Cloudflare Workers）

### プロパティ一覧

| プロパティ名 | Notion型 | 備考 |
|---|---|---|
| 日付 | date | YYYY-MM-DD |
| 大項目 | title（タイトル） | カテゴリ |
| 会社名 | multi_select | 複数選択可 |
| 開始時間 | date（時刻あり） | ISO形式・タイムゾーン付き |
| 終了時間 | date（時刻あり） | ISO形式・タイムゾーン付き |
| 工数(h) | number | 小数（例: 1.5） |
| 作業内容 | rich_text | 作業の詳細 |

> **数式プロパティで自動計算可能**:
> `dateBetween(prop("終了時間"), prop("開始時間"), "hours")`

## localStorage キー一覧

| キー | 内容 |
|---|---|
| `work_logs` | 作業ログ（JSON配列） |
| `work_cats` | 大項目リスト（JSON配列） |
| `work_companies` | 中項目・会社リスト（JSON配列） |
| `notion_token` | Notion Integration Token |
| `notion_db_id` | Notion Database ID |
| `last_open_date` | 前回起動日（YYYY-MM-DD）。日付変更検知に使用 |
| `skip_delete_confirm` | `"true"` のとき日付変更時に確認なし自動削除 |

## 実装済み機能

### 入力フォーム
- **大項目**（select）: localStorage に保存、UI から追加・名前変更・削除可能
- **中項目・会社**（チェックボックス）: 複数選択可、localStorage に保存、UI から追加・名前変更・削除可能
- **記録ボタン**: 1回目クリック→現在時刻で開始時間をセット、2回目クリック→終了時間をセット（ボタンが赤く点滅）
- **開始＝今(丸) / 終了＝今(丸)**: 30分単位に丸めてセット
- 時間は手動入力も可能

### 起動時クリーンアップ
- アプリ起動時に `last_open_date` と今日を比較し、日付が変わっていれば前日以前のログを削除するか確認モーダルを表示
- モーダルで「次回以降確認しない」にチェックすると、次回以降は確認なしで自動削除（`skip_delete_confirm` に保存）
- 「残す」を選んだ場合は削除しない

### ログ管理
- ログは日付をまたいでも削除されず、全期間分を保持
- 新規追加時にNotion設定済みなら**自動でNotionへ送信**
- Notion送信ステータスをアイコンで表示（✓ 送信済 / ! エラー / − 未送信）
- 削除は確認ダイアログあり
- **編集機能**: ログ行の ✎ ボタンでフォームにデータを呼び出し、修正して保存できる
  - 日付・大項目・会社・時間・内容をすべて変更可能
  - Notion送信済みのログは PATCH API で上書き更新
  - 未送信のログは編集保存後に新規送信

### Notion連携
- **自動送信**: ログ追加時に即座に送信
- **編集時更新**: ログ編集保存時に PATCH API（`/v1/pages/{id}`）で該当ページを上書き
- **未送信を再送信**: 失敗したものだけ再送
- **今日分を取り込む**: NotionのDBに当日日付で登録されているレコードのうち、ローカルにないものだけ取り込む（スマホで追加したものをPCに同期する用途）

### サマリー
- 今日分のみ集計（総工数・件数・カテゴリ別バー）

### 日報プレビュー
- kintone 向けテキスト自動生成
- 会社名も含めてフォーマット: `[09:00〜10:30]（1.5h）開発・実装 [会社A]`
- クリップボードへのコピー機能

## 技術的な判断メモ

- **開始時間・終了時間を date 型にした理由**: Notionの数式・フィルタ・タイムラインビューを活用するため（テキスト型では不可）
- **会社名を multi_select にした理由**: 複数社に対応、Notionでの絞り込みに使えるため
- **日次リセットを廃止した理由**: データが消えることがあったためログを永続保持に変更
- **プロキシを使う理由**: ブラウザからNotionAPIに直接アクセスするとCORSエラーになるため
- **ログIDはDate.now()**: 一意性確保のためタイムスタンプをIDに使用
